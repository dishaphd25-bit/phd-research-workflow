Traceability: Git provides a clear timeline of every change, making research auditable.
Collaboration: GitHub allows multiple researchers to share code and track issues globally.
Branches: Creating branches keeps the main branch stable while experimenting with new data or models.
Pull Requests: They allow for code review and discussion before changes are permanently merged.
Tags: Tags like v1.0.0 mark specific stable states, such as a version used for a paper submission.
