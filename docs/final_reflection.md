Traceability: Git provides a clear timeline of every change, making research auditable.
Collaboration: GitHub allows multiple researchers to share code and track issues globally.
Branches: Creating branches keeps the main branch stable while experimenting with new data or models.
Pull Requests: They allow for code review and discussion before changes are permanently merged.
Tags: Tags like v1.0.0 mark specific stable states, such as a version used for a paper submission.



1.	How does Git help in maintaining research traceability? 
	Git records every change in a timeline, allowing researchers to audit and revert to any previous state of their work.
2.	Why is GitHub useful for academic collaboration? 
	It provides a centralized platform for multiple researchers to share code, track experimental issues, and review each other's work.
3.	Why should researchers create branches instead of working directly on main? 
	Branches isolate new experiments or documentation tasks, keeping the main branch stable for "publication-ready" code.
4.	What is the role of Pull Requests in collaborative work? 
	They act as a formal review phase where code and assumptions are discussed before being integrated into the project.
5.	Why are reproducibility files important in research repositories? 
	Files like requirements.txt and checklists ensure other scientists can accurately recreate your environment and results.
6.	What is the purpose of creating a version tag such as v1.0.0? 
	A tag marks a specific, immutable version of the code used for a thesis submission or a peer-reviewed paper.
7.	Why should a repository include a citation file? 
	It ensures the researcher receives credit for their work and makes the repository easily reusable in academic literature.
8.	How does this semester's Git/GitHub workflow support better PhD research practice? 
	It builds a habit of version-controlling code, data, and documentation together, leading to more transparent and credible research.
